# Metamorphic Eval

## Project Structure

```
thesis-metamorphic-eval/
├── agents/
├── benchmarks/
    ├── downloads/
├── code-coccoon/
├── repos/
├── results/
├── scripts/
    ├── transform.py
    ├── bootstrap.py
├── swe_bench/
├── transformations/
```

### Purpose

1. `agents`: The [bootstrap.py](./scripts/bootstrap.py)'s `init` subcommand downloads the [MSWE-agent](https://github.com/Vladislav0Art/MSWE-agent) into this folder.
1. `benchmarks`:
    1. `./downloads`: Multi-SWE-bench and Multi-SWE-bench_mini are downloaded here by the [bootstrap.py](./scripts/bootstrap.py)'s `benchmark` subcommand.
    1. The folder itself is intended to contain JSONL files with the benchmark definitions (see example in [sample.jsonl](./benchmarks/sample.jsonl))
1. `code-coccoon`: The [bootstrap.py](./scripts/bootstrap.py)'s `init` subcommand downloads the [CodeCocoon-Plugin](https://github.com/JetBrains-Research/CodeCocoon-Plugin) into this folder.
1. `repos`: The [transform.py](./scripts/transform.py) script will download repositories from the benchmarks JSONL file into this folder and run CodeCocoon's transformations on them.
1. `results`: Thesis results are intended to be stored here.
1. `scripts`: scripts are store in this folder ([transform.py](./scripts/transform.py), [bootstrap.py](./scripts/bootstrap.py))
1. `swe_bench`: The [bootstrap.py](./scripts/bootstrap.py)'s `init` subcommand downloads the [multi-swe-bench](https://github.com/Vladislav0Art/multi-swe-bench) into this folder.
1. `transformations`: JSON files with transformations' definitions for the [codecoccoon.yml](https://github.com/JetBrains-Research/CodeCocoon-Plugin/blob/main/codecocoon.example.yml) files generated by [transform.py](./scripts/transform.py) are store in this folder (see [sample.json](./transformations/sample.json)).

## Getting Started

### Workflow

The general workflow:

1. **Preparation**:
    1. Initialize repositories (_CodeCocoon-Plugin, MSWE-agent, multi-swe-bench_) (*)
    1. Install the needed (sub)set of benchmarks as a JSONL file (_**Multi-SWE-Bench** or **Multi-SWE-Bench-mini** supported_).
    1. Generate CodeCocoon's metamorphic transformations and insert their diffs into the benchmarks JSONL file.
1. **Execution**:
    1. Execute MSWE-agent on both versions (non-metamorphic and metamorphic) of the benchmarks (_the matter of applied patches to the base commit of each benchmark_).
1. **Evaluation**:
    1. Run **multi_swe_bench** repo's evaluation script to get final eval metric.
    1. [**Additionally**] Given the agent's output on non-metamorphic and metamorphic diffs, calculate some other metrics (_not yet implemented_).


### Prerequisites

1. Create virtual environment:
```bash
python3 -m venv ./venv
source ./venv/bin/activate
```

2. Install packages:
```bash
pip install -r requirements.txt
```



### Preparation

The `bootstrap.py` script sets up the evaluation environment by downloading all 3 repos (*) and collecting a set of benchmarks as a single JSONL file.

#### 1. Initialization

Downloads required repositories:

```bash
python scripts/bootstrap.py init
# Or with custom URLs:
python scripts/bootstrap.py init --codecoccoon <url> --mswe_agent <url> --multi_swe_bench <url>
```

Run the help command to see the default URLs:
```bash
python scripts/bootstrap.py init --help
```


#### 2. Benchmarks

Creates filtered benchmark datasets:
```bash
# Basic usage
python scripts/bootstrap.py benchmark Multi-SWE-bench_mini --language java --difficulty easy

# With random sampling (selects 10 random out of all JSONL entries)
python scripts/bootstrap.py benchmark Multi-SWE-bench --language java --difficulty medium --rand 10

# With custom output path
python scripts/bootstrap.py benchmark Multi-SWE-bench_mini --language python --output my-benchmark.jsonl
```

Run the help command to see the supported values of every parameter:

```bash
python scripts/bootstrap.py benchmark --help
```

#### 3. Metamorphic Transformations

The [transform.py](./scripts/transform.py) executes the following algorithm:

1. Accepts the set of benchmarks collected by the [bootstrap.py](./scripts/bootstrap.py)'s `benchmark` subcommand.
1. For every benchmark, clones its repo into `repos/{benchmark.instance_id}` and runs [CodeCocoon-Plugin](https://github.com/JetBrains-Research/CodeCocoon-Plugin) that applies transformations from the JSON file passed as a filepath via `-t/--transformations` parameter (see [sample.json](./transformations/sample.json)).
1. The diffs after applying metamorphic transformations will be saved into every benchmark's JSON entry and saved as a new JSONL file under a filepath passed as `-o/--output` parameter. The schema will be:

```json
{
    // ... other benchmark entries ...
    "metamorphic": {
        "[strategy-name]": {
            "logs": [
                {
                    "applied_to": "base",
                    "label": "metamorphic_base_patch_log",
                    "result": {
                        "stdout": "STDOUT of the CodeCocoon-Plugin executed on this benchmark on BASE commit",
                        "stderr": "STDERR of the CodeCocoon-Plugin executed on this benchmark on BASE commit",
                        "return_code": 0
                    }
                },
                {
                    "applied_to": "fix",
                    "label": "metamorphic_fix_patch_log",
                    "result": {
                        "stdout": "STDOUT of the CodeCocoon-Plugin executed on this benchmark AFTER applying BOTH fix and test patches",
                        "stderr": "STDERR of the CodeCocoon-Plugin executed on this benchmark AFTER applying BOTH fix and test patches",
                        "return_code": 0
                    }
                }
            ],
            "metamorphic_base_patch": "diff --git a/path/to/file.java b/path/to/file.java\n ...",
            "metamorphic_fix_patch": "diff --git a/path/to/file.java b/path/to/file.java\n ..."
        }
    }
}
```

Where `strategy-name` is passed as a `-s/--strategy` parameter of the [transform.py](./scripts/transform.py) script.


Run the help command to see the list of available parameters and their purpose:
```
python scripts/transform.py --help
```

**General usage**:

Given filepaths will be converted to absolute filepaths.

Here is a runnable example:

```bash
python scripts/transform.py -i ./benchmarks/sample.jsonl \
    -o ./benchmarks/metamorphic-sample.jsonl \
    -s 'MyStrategy' \
    -c ./code-coccoon/CodeCocoon-Plugin \
    -r ./repos \
    -t ./transformations/sample.json \
    --override
```

After running, navigate to `benchmarks/metamorphic-sample.jsonl` and search for `"metamorphic"` JSON field.
